# CodeFRAME — Product Requirements

This file used to describe a v1 product: an autonomous system where specialized agents
(lead, backend, frontend, test, review) collaborated to write code. **That product was
abandoned.** CodeFRAME no longer writes code — it orchestrates the frontier coding
agents that do, and owns the edges of the pipeline: Think → Build → Prove → Ship.

Product requirements now live in:

| Document | Purpose |
|----------|---------|
| [`docs/VISION.md`](docs/VISION.md) | North star — the Think → Build → Prove → Ship thesis |
| [`docs/PRODUCT_ROADMAP.md`](docs/PRODUCT_ROADMAP.md) | Current phase plan and shipped features |
| [`docs/GOLDEN_PATH.md`](docs/GOLDEN_PATH.md) | The CLI-first workflow contract |
| [`docs/CLI_WIREFRAME.md`](docs/CLI_WIREFRAME.md) | Command tree → core module mapping |
| [`CLAUDE.md`](CLAUDE.md) | Current status, architecture rules, working mode |

The v1 requirements this file used to hold remain in git history (`git log -- PRD.md`).
Other v1-era documents are archived under [`legacydocs/`](legacydocs/). Neither
describes the current product.
